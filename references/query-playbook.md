# Query Playbook

Use this playbook to start broad news collection quickly without improvising every query from scratch.

## How to use this playbook

For each topic bucket:

1. Start with one `core` query
2. Add one `news` query for recency
3. Add one `institutional` or `company` query when evidence strength matters
4. If coverage feels thin, run one Chinese query and one English query before widening further

Prefer 2 to 4 strong queries per bucket over a long pile of weak ones.

Do not rely only on search queries. For recurring work, pair your queries with at least one explicit source-entry style:

1. `hotlist entry` — ranked or trending boards for fast discovery
2. `official entry` — regulator, company, standards body, exchange, or institution page
3. `community entry` — HN, GitHub, Reddit, or other practitioner-discussion source
4. `watchlist entry` — named companies, projects, repos, ministries, or tickers

## Global query frames

Use these wrappers across topics:

1. `latest <topic> news`
2. `<topic> developments last 7 days`
3. `<topic> policy OR regulation OR guidance`
4. `<topic> funding OR acquisition OR earnings`
5. `<topic> official statement`
6. `<topic> market trend`
7. `<topic> 中国`
8. `<topic> 全球`

## Query mixing rules

When the briefing is broad:

1. Pair one Chinese query with one English query for each advanced technology topic
2. Pair one market query with one policy query for regulated sectors
3. Pair one general-news query with one official-source query for politics or public policy
4. For entertainment and sports, pair one headline query with one platform or league query
5. For academic topics, pair one discovery query with one source-database query
6. For capital markets, pair one market query with one disclosure or regulator query
7. For energy and power topics, pair one policy query with one company or project query

## Escalation rules

Escalate to deeper verification if:

1. The same event appears with conflicting numbers or dates
2. A story could affect regulation, capital markets, or safety decisions
3. A viral story appears to rely on one origin source
4. The topic is technical enough that headlines may distort the underlying claim
